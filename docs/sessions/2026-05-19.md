## Bugs Fixed

- **Drive mutations 403 (file not found)** — missing `supportsAllDrives: true` on permission delete/update/create for Shared Drive files
- **Inherited permission silent failure** — Google 403 was surfaced as `alert()`; now shows actionable snackbar with drive name
- **Wrong drive name in error** — `drives.get` missing `useDomainAdminAccess: true`; fallback template also grammatically broken
- **My Drive permissions empty in modal** — `useDomainAdminAccess: true` on `permissions.list` causes Google to return `[]` for non-Shared-Drive files; now conditional
- **Stale modal permissions/path** — `handleOpenPermissionDialog` passed list-state directly; now re-fetches `GET /drive/files/:id` on open
- **Email wrapping** — `wordBreak: break-word` replaced with nowrap + ellipsis + Tooltip
- **Export CSVs missing external emails** — four export routes had duplicated inline classification logic with bugs; replaced with `classifyExternalSharing`

## Pending / Carry-over

- **Path still shows drive root only in file list** — `buildFastPath` intentionally only returns `/Shared Drives/<name>` (no subfolders) for performance during bulk scan. The modal now shows the correct full path. If full path in list view is desired, it would require per-file API calls during scan (significant perf cost) or a lazy-load on hover/expand.
- **Confirm dialogs** — several actions still use `window.confirm()` (delete permission, bulk remove). Should be replaced with MUI `Dialog` components for consistent UX.
- **`alert()` in `handleUpdatePermission`** — still uses `alert()` on failure (line ~591). Should be converted to `setSnackbar`.
- **`alert()` in `handleOpenPermissionDialogForReport`** — still uses `alert()` on failure. Should be converted to `setSnackbar`.
